行业打标 · 用户全链路闭环

用户 → 飞书 → Agent → SQL+Python+Skill+LLM → 飞书 → 用户

Python 脚本

  • 从 SR 查询原始数据（公司名/经营范围）
  • 关键词提取：局/银行/科技/智能 等
  • 数据清洗、格式化、缓存
  • 生成 LLM 可用的结构化输入
  • 纯数据处理，不需要智能判断

用户交互节点

  • 入口：用户通过飞书消息发指令给 Agent
  • 中间：Agent 完全自动执行，用户无需介入
  • 出口：Agent 完成打标后通知用户
  • 用户去飞书表格查看/确认结果
  • 用户可继续发消息处理下一批

Agent 编排

  • 收消息 → 决定 SQL 查 SR
  • → 调 Python 特征提取
  • → 加载 Skill 分类标准
  • → 构造 Prompt 调 LLM
  • → lark-cli 写回飞书
  • → 飞书消息通知完成
